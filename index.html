RESTful-Message-Beans by explainer

demo rails app that uses daemons to route message to controller

Uses recipes from 'Enterprise Recipes with Ruby and Rails' to build a 'message-bean' style of receiving broker messages in a rails app. A Listener model object is created for each topic or queue. When a listener is started, a daemon is created to listen at the broker and to use http to post the message to the documents_controller. The message ends up in the 'data' field of the documents table.

Dependencies

many gems used in the book, ActiveMQ used as message broker, MySQL as DB

Install

Install and configure git on your local machine.
